# 📈 Production-Grade AI Trading Infrastructure & Dashboard

A containerized, low-latency automated trading system prototype built on **Python 3.12**, leveraging **Streamlit** for real-time visualization and **FastAPI** for an asynchronous backend loop execution daemon. The framework isolates machine learning inference pipelines and algorithmic strategies, creating an AI-friendly interface designed to support clean drop-in code generated by LLMs like Claude or Gemini.

---

## 🗺️ Microservice Communication Architecture Flow

The system is decoupled into two independent microservices to guarantee separation of concerns. If the UI container or client web browser drops a connection, the backend execution daemon remains unaffected, safely managing active orders and state variables inside the cluster.

```text
       ┌──────────────────────────────┐              ┌──────────────────────────────┐
       │   Streamlit Web Dashboard    │              │    FastAPI Execution Worker  │
       │   Container (Port 8501)      │              │    Container (Port 8080)      │
       └──────────────┬───────────────┘              └──────────────┬───────────────┘
                      │                                             │
                      │  [1] GET /api/state (Poll State & Logs)     │
                      ├────────────────────────────────────────────►│
                      │  [2] POST /api/state/toggle (Start/Stop)    │
                      ├────────────────────────────────────────────►│
                      │                                             │
                      │                                   Spawns Asynchronous Task
                      │                                             │
                      │                                             ▼
                      │                              ┌──────────────────────────────┐
                      │                              │   Continuous Async Loop      │
                      │                              │   - Poll Broker Data & Ticks │
                      │                              │   - Execute ML Inference     │
                      │                              │   - Run Pluggable AI Strategy│
                      │                              │   - Route Validated Orders   │
                      │                              └──────────────┬───────────────┘
                      │                                             │

       ┌──────────────────────────────┐              ┌──────────────────────────────┐

```
## 🔒 Multi-Thread Validation Safety Blueprint

To securely handle concurrent transactions, the system routes all active calculations, operational logs, and position matrices through a unified, thread-safe memory manager (`core/state_manager.py`). 

Because the backend execution loop updates market data on one frequency while the FastAPI thread pools constantly service external HTTP read requests from the Streamlit UI, standard library mutual exclusion primitives (`threading.Lock`) are used to completely eliminate memory corruption risks and data race conditions.

```text
   [ FastAPI Request Thread Pool ]                [ Asynchronous Core Worker Loop ]
                  │                                               │
                  ▼                                               ▼
     (Reads Account Analytics)                        (Ingests Real-Time Tick Feeds)
        get_summary_metrics()                           update_market_data()
                  │                                               │
                  └──────────────► [ Acquires Thread Lock ] ◄─────┘
                                                  │
                                    Modifies system memory safely
                                    without causing data race conditions
                                                  │
                                                  ▼
                                      [ Releases Thread Lock ]
```

### Key Telemetry Guarantees:
* **Atomic Telemetry Queries**: The visual web dashboard will never read a halfway-mutated data struct while a live broker websocket thread processes an active price tick.
* **Isolated State Mutation**: Strategy modules drop in code without directly altering database arrays, communicating structural trade signals cleanly via standardized dictionaries.

## 🛠️ Local Workspace Development (openSUSE Tumbleweed & PyCharm)

### 1. System Requirements & Dependencies
Before installing the Python requirements, you must compile or install the underlying `TA-Lib` C-library headers on your host openSUSE machine:
```bash
sudo zypper ref
sudo zypper in ta-lib-devel build-essential
```

### 2. PyCharm Virtual Environment Configuration
1. Open PyCharm and configure a new project environment using **Python 3.12**.
2. Run the native initialization package script inside the PyCharm terminal to sync dependencies:
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 3. Local Local Manual Execution
To run and test the decoupled pipeline without orchestration, execute both microservices in separate shell panels:
```bash
# Terminal 1: Launch Backend Engine Daemon Process
python main.py

# Terminal 2: Launch Visual UI Front-end Application
streamlit run dashboard/app.py
```

Build the microservices locally using optimized multi-stage build scripts to strip compilation dependencies and maintain a minimal memory footprint:

```bash
# Build interactive visualization engine front-end image
docker build -t trading-dashboard:latest -f Dockerfile.dashboard .
```

## ☸️ Cloud Deployment Configuration (Kubernetes)

To place the microservices into a live environment (AWS EKS, GCP GKE, or a local k3s instance), navigate into your repository root directory and apply the declarative manifests:

```bash
# 1. Establish isolation context layers
kubectl apply -f deploy/namespace.yaml

# 2. Inject global configurations and secure credentials
kubectl apply -f deploy/configmap.yaml
kubectl apply -f deploy/secrets.yaml

# 3. Mount singleton execution daemon and scalable UI layers
kubectl apply -f deploy/backend-deployment.yaml
kubectl apply -f deploy/dashboard-deployment.yaml
```

### ⚠️ Critical Scalability Rule
The execution backend deployment manifest `deploy/backend-deployment.yaml` is locked strictly to `replicas: 1`. **Do not alter this parameter.** Running parallel instances of an asynchronous execution loop using the same API keys can cause race conditions, resulting in double-executions and out-of-order broker allocations.
